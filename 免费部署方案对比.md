## 🥇 方案1: Vercel + 远程MySQL (强烈推荐!)

### ✅ 优势
- **完全免费**,不需要信用卡
- **速度快**,全球CDN加速
- **自动HTTPS**,安全可靠
- **24小时运行**,不占用电脑
- **适合7-8人**同时使用
- **一键部署**,非常简单

### ❌ 限制
- 前端托管在Vercel,后端需要单独部署
- 需要使用免费的远程MySQL服务

### 📋 所需准备
1. GitHub账号(免费)
2. Vercel账号(免费,用GitHub登录)
3. 免费MySQL数据库服务(选择其一):
   - **PlanetScale** (推荐): https://planetscale.com/ - 5GB免费,不需要信用卡
   - **TiDB Cloud**: https://tidbcloud.com/ - 5GB免费,不需要信用卡
   - **Aiven**: https://aiven.io/ - 免费额度,不需要信用卡

### 🚀 部署步骤

#### 第1步: 准备数据库 (5分钟)
1. 访问 PlanetScale: https://planetscale.com/
2. 用GitHub账号登录
3. 点击 "Create database"
4. 选择免费套餐
5. 创建完成后,复制连接信息:
   - Host
   - Username
   - Password
   - Database name

#### 第2步: 修改后端配置 (2分钟)
编辑 `backend/.env` 文件:
```env
MYSQL_HOST=你的PlanetScale主机地址
MYSQL_PORT=3306
MYSQL_USER=你的用户名
MYSQL_PASSWORD=你的密码
MYSQL_DATABASE=shophub
PORT=3000
```

#### 第3步: 推送到GitHub (3分钟)
```powershell
cd C:\Users\L2604\IdeaProjects\untitled1\merchant collection
git init
git add .
git commit -m "准备部署到Vercel"
git remote add origin https://github.com/你的用户名/shophub.git
git push -u origin master
```

#### 第4步: 部署到Vercel (5分钟)
1. 访问: https://vercel.com/
2. 用GitHub账号登录
3. 点击 "New Project"
4. 导入你的GitHub仓库
5. 点击 "Deploy"
6. 等待1-2分钟,完成!

#### 第5步: 获取访问地址
Vercel会给你一个地址: `https://shophub.vercel.app`

分享给团队成员即可使用!

## 🥈 方案2: Railway (推荐!)

### ✅ 优势
- **$5免费额度/月**(足够小项目使用)
- **不需要信用卡**注册
- **全栈部署**,前后端一起部署
- **自带MySQL数据库**
- **24小时运行**

### ❌ 限制
- 每月$5额度,超出后需要付费
- 对于7-8人小团队,免费额度通常够用

### 📋 部署步骤

#### 第1步: 注册Railway (2分钟)
1. 访问: https://railway.app/
2. 用GitHub账号登录
3. 不需要绑定信用卡

#### 第2步: 创建项目 (3分钟)
1. 点击 "New Project"
2. 选择 "Deploy from GitHub repo"
3. 选择你的仓库
4. Railway会自动检测Node.js项目

#### 第3步: 添加MySQL数据库 (2分钟)
1. 在项目中点击 "+ New"
2. 选择 "Database" → "Add MySQL"
3. 等待数据库创建完成

#### 第4步: 配置环境变量 (2分钟)
在Railway面板中设置:
```
MYSQL_HOST=${{MySql.MYSQLHOST}}
MYSQL_PORT=${{MySql.MYSQLPORT}}
MYSQL_USER=${{MySql.MYSQLUSER}}
MYSQL_PASSWORD=${{MySql.MYSQLPASSWORD}}
MYSQL_DATABASE=${{MySql.MYSQLDATABASE}}
PORT=3000
```

#### 第5步: 部署 (3分钟)
1. 点击 "Deploy"
2. 等待部署完成
3. 获取访问地址

## 🥉 方案3: Render (可用)

### ✅ 优势
- **完全免费**,不需要信用卡
- **全栈部署**
- **简单易用**

### ❌ 限制
- 免费套餐有性能限制
- 首次访问可能需要等待启动(冷启动)
- 速度相对较慢

### 📋 部署步骤

#### 第1步: 注册Render (2分钟)
1. 访问: https://render.com/
2. 用GitHub账号登录

#### 第2步: 创建Web Service (5分钟)
1. 点击 "New Web Service"
2. 连接GitHub仓库
3. 选择免费套餐
4. 配置环境变量(Mysql连接信息)

#### 第3步: 添加数据库 (5分钟)
1. 点击 "New PostgreSQL" 或 "New MySQL"
2. 选择免费套餐
3. 复制数据库连接信息

#### 第4步: 部署 (3分钟)
1. 配置完成后点击 "Create Web Service"
2. 等待部署完成

## 🔧 方案4: 本地电脑 + 内网穿透 (临时方案)

### ✅ 优势
- **完全免费**
- **不需要信用卡**
- **数据在本地**,安全可控
- **部署简单**

### ❌ 限制
- **需要你的电脑一直开机**
- 依赖你的网络稳定性
- 速度受限于你的宽带上传速度

### 📋 所需工具
- **ngrok** (免费): https://ngrok.com/
- 或者 **frp** (开源): https://github.com/fatedier/frp

### 部署步骤

#### 第1步: 本地启动应用
```powershell
cd C:\Users\L2604\IdeaProjects\untitled1\merchant collection
npm install
node backend/server.js
```

#### 第2步: 安装ngrok
```powershell
# 下载ngrok
# 访问: https://ngrok.com/download
# 或使用choco安装
choco install ngrok
```

#### 第3步: 启动内网穿透
```powershell
ngrok http 3000
```

会生成一个公网地址: `https://xxxx.ngrok.io`

#### 第4步: 分享地址
将ngrok生成的地址分享给团队成员

⚠️ **注意**: 免费版的ngrok地址每次重启会变化
